1 - Vision deficiencies

In Dev Tools open ⠇ menu > More tools > Rendering and use the Emulate vision deficiencies drop-down to identify the problems with the following graphic:

Answer 🎒 list color-related problems you see with the graphic above:
  • Deuteranopia -- People would be unable to differntiate the color scale as for example both the extremes (dark red and dark green) look the same color (dark brown)
  • Deuteranopia -- The extremes are slighlty better but still unable to differntiate all the colors as diffent values in the scale still have the same color
  • Tritanopia -- The color flips i.e red becomes green, which would give a completely different meaning to the data. (i.e large values becomes small and small values become large)
  • Text size should be larger, as people, particularly with blurred vision will be unable to read
  • Size of the map should be a little larger, to soo the color differences in the top middle part more clearly

2 - Color accessibility

Use Dev Tools as in Accessible Color Standards - Designing in the Browser to correct the color contrast where marked with 🎒.

💡 WebAIM color accessibility levels: Level A: basic requirement for some users with disabilities to be able to access and use web content -- color contrast of 3:1
Level AA: overall accessibility and removal of significant barriers to accessing content -- color contrast of 4.5:1
Level AAA: provides improvements and enhancements to web accessibility for some users with disabilities -- color contrast of 7:1

Answer

🎒 adjust text color so the color contrast with the background complies with AA level
